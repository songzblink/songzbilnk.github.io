ansible主机登录认证

ansible主机组在/etc/ansible/hosts文件中配置。

ansible批量管理主机需要登录到目标主机，而登录认证有两种方式：

  • 传统的密码认证
  • 公钥认证

传统的密码认证即通过密码进行登录，需要配置ansible_host(主机ip地址)、ansible_port(主机ssh端口)、ansible_user(主机用户)、ansible_password(主机用户密码)。

公钥认证可以省略每次登录都需要输入密码的问题，其流程为：

  1. 主机A生成公钥后，将公钥发送给其他主机；
  2. 主机A在登录其他主机的过程中，其他主机会返回一个随机字符串，主机A利用私钥对这个随机字符串进行加密再发往想要登录的主机，如果目标主机将加密后的随机字符串用公钥解密正确，就允许主机A登录，否则拒绝。

也就是说公钥分发后就可以直接利用ansible ad-hoc模式去管理目标主机。

所以，如果我们想要使用ansible管理其他主机，最方便的方法还是进行公钥认证，即先分发公钥实现免密登录，再用ansible免密远程执行命令。

模块文档可在这里找到：Ansible.Builtin — Ansible Documentation

ansible执行命令结果（状态颜色）

ansible执行命令的结果会有不同的颜色，这些颜色有各自的含义。

  • 绿色：命令以用户期望的形式执行了，但是状态没有发生变化
  • 黄色：命令以用户期望的形式执行了，并且状态发生了变化
  • 紫色：警告信息，说明ansible提示你有更合适的用法
  • 红色：命令错误，执行失败
  • 蓝色：详细的执行过程

command模块

command时ansible的基础模块，默认携带-m command。

，使用command模块执行远程命令时，命令不能使用变量（${HOME})，也不能出现特殊符号（<，>，|，:，&），否则将无法识别，一定需要使用的话可以用shell模块实现。即command模块不能使用复杂的linux命令。

可以在-a后的字符串内增加配置选项“warn=false"来关闭告警信息：

在使用command命令时，还能够使用专有命令，这些命令用于编写ansible-playbook，完成服务器部署的各种复杂条件限定。

选项参数 选项说明
chdir=目录路径 在执行命令时执行，通过cd命令进入指定目录
creates 定义一个文件是否存在，若不存在，则运行相应命令；存在则跳过
free_form（必须） 参数信息中可以输入任何系统命令，实现远程管理
removes=文件路径 定义一个文件是否存在，如果存在则运行相应命令，如果不存在则跳过

例如cd / && tar -zcvf /opt/log.tar /var/log

shell模块

shell模块相当于万能模块，尽管它可以让你在目标linux机器上运行任何命令，但还是建议使用前先查看是否有专有的模块，这样能够利用ansible能够记录状态的功能。

copy模块

copy模块是远程推送数据模块，只能把数据推送给远程主机节点，无法拉取数据到本地

参数名 是否必须 默认值 选项 说明
src no 用于定位ansible执行的机器上的文件，需要绝对路径。如果拷贝的是文件夹，那么整个文件夹会一起拷贝过去，如果结尾是“/”，那么只有文件夹内的东西会被拷过去。
content no 用来替代src，用于指定文件的内容，拷贝到远程文件内
dest yes 用于定位远程节点上的文件，需要绝对路径。如果src指定的是文件夹，这个参数也必须是文件夹
backup no no yes/no 备份远程节点上的原始文件
directory_mode no 该参数只能用于拷贝文件夹时，该参数设定后，文件夹内新建的文件会被拷贝，旧的则不会
follow no no yes/no 当拷贝的文件夹内有link存在的时候，拷贝过去的也会有link
force no yes yes/no 默认为yes，会覆盖远程的内容不一样的文件（即使文件名一样）。如果是no，如果远程有这个文件就不会进行拷贝。
group no 设定一个群组拥有拷贝到远程节点的文件权限
mode no 等同于chmod
owner no 设定一个用户拥有拷贝到远程节点的文件权限

ansible会记录文件属性，得到文件的唯一校验值，判断文件内容是否变化，如果未变化则不做处理，提升批量管理的效率。

free_form（必须） 参数信息中可以输入任何系统命令，实现远程管理
removes 定义一个文件是否存在，若存在则运行相应命令，若不存在则跳过

模块参数 参数说明
daemon_reload 在执行任何其他操作之前运行守护进程重新加载，以确保systemd已经读取其他更改

masked 是否将服务设置为masked状态，被mask的服务无法启动
name 服务名称，必填
no_block 不要同步等待操作请求完成
state started，启动；stopped，停止；restarted，重启；reloaded，重新加载
user 使用服务的调用者运行systemctl，而不是系统的服务管理者
